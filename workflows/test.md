# Test Workflow

## Prerequisites

- [ ] Component code exists.

## Steps

1. Identify the component you want to test.
2. Create a new test file or update existing tests.
3. If the test fails, fix the code and run the test again until it passes.
4. Ensure the test structure mirrors the code structure.

## Exit Criteria

- [ ] Test file exists in correct location.
- [ ] Tests pass (or fail as expected for TDD).
